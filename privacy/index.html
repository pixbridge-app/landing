Privacy Policy

Effective Date: July 14, 2026

This Privacy Policy explains how PixBridge handles information in connection with the mobile app, access requests, and related pages.

1. Overview

PixBridge is a mobile Camera-to-API client. The app helps users capture images on a mobile device, send those images to a server API configured by the user, and display selected fields from the JSON response on the app screen.

This policy explains the data practices that may apply when users access PixBridge and related services.

2. Information We May Collect

Account information

PixBridge uses Firebase Authentication for account sign-in and account management. PixBridge may collect information associated with authentication, account identifiers, and user settings.

Server configuration data

Users may save server addresses, request or transmission format settings, image field names, response key settings, labels, types, and related app configuration. These settings may be stored in Cloud Firestore and linked to the user's account to support synchronization or reuse.

Optional authentication tokens entered by the user

Users may optionally enter Bearer tokens or similar authentication values for their own APIs. These values are user-provided credentials and should be treated carefully by the user.

Images captured and sent by the user

Users may capture or select images through the mobile app. During normal app use, PixBridge sends those images to the server API configured by the user.

API response data

PixBridge may process the JSON response returned by the user's configured API in order to display selected keys or fields in the app. Response setting data may include key names, labels, data types, and default display values.

Access request and contact form submissions

Access request or contact forms may collect information such as name, email address, organization, intended use case, and message details. These submissions may be used to respond to requests, provide service information, and improve PixBridge.

Basic technical and usage data

PixBridge or its service providers may process basic technical information such as device or app version, request status, timing, response size, and similar data needed for operation and improvement.

Google Analytics for Firebase data

PixBridge uses Google Analytics for Firebase. Firebase Analytics is enabled by default when the app is installed and used. The Firebase SDK may automatically collect events such as first_open, session_start, and user_engagement, and PixBridge may log additional custom events for major feature flows.

Analytics data may include app launch and session information, app feature usage and interaction information, screen or feature visits, app version, operating system, device type, language, technical identifiers used to distinguish app installation instances, subscription page visits and entitlement activation status, image send success or failure, non-identifying failure categories, and feature events such as server creation, server switching, preset use, import, and export.

PixBridge uses allowed category values and aggregate values for custom Analytics events. PixBridge does not include the following in custom Analytics event parameters: actual server URLs, hostnames, endpoint paths, query strings, Bearer tokens, API keys, captured images, image file names or file paths, raw response JSON, response bodies, user-defined response keys, labels, values, email addresses, user names, Firebase uid values, IP addresses directly recorded by PixBridge as custom event parameters, server names, preset names, free-text strings, exception messages, stack traces, transaction IDs, or receipts.

Payment and subscription information

PixBridge may use RevenueCat and app store payment providers to manage purchases, subscriptions, entitlement status, refunds, and related payment events. PixBridge does not directly receive full payment card details from users.

3. How We Use Information

  • To provide the core Camera-to-API workflow.
  • To save and reuse server configurations and response settings.
  • To display selected JSON response fields in the app.
  • To support account, authentication, synchronization, and service communication.
  • To manage purchases, subscriptions, entitlements, refunds, and payment-related support.
  • To analyze app usage, including the flow from new sign-up to server creation, image sending, and record review.
  • To analyze image send success rates and non-identifying error categories.
  • To analyze subscription page visits and actual plan activation flows.
  • To improve feature usability, performance, app stability, and user experience.
  • To investigate bugs, maintain reliability, improve the service, and respond to user requests.
  • To help protect the service, users, and related systems from misuse.

PixBridge does not use Analytics data for advertising, user profiling for advertising, or personalized advertising.

4. Image and API Data Handling

PixBridge sends captured or selected images to the server API configured by the user. PixBridge does not provide image analysis services on its own server during normal app usage.

The API response is returned by the user's configured server. PixBridge may read selected JSON keys from that response and display them on the mobile screen according to the user's response settings.

Users should avoid sending images containing personal information, sensitive information, faces, vehicle plates, identity documents, precise location details, or confidential materials unless they have the appropriate rights, permissions, and safeguards.

5. User-Configured Servers and Third-Party APIs

Because PixBridge sends images to the API endpoint configured by the user, the storage, analysis, retention, and security practices of that external server are controlled by the user or the server/API operator.

Users are responsible for reviewing the privacy and security practices of any server or third-party API they connect to PixBridge.

6. Third-Party Services

PixBridge may use third-party services to support app operation, hosting, communication, account-related features, and purchases. These may include:

  • Google Analytics for Firebase for app usage measurement and feature flow analysis.
  • Firebase Authentication for sign-in and account management.
  • Cloud Firestore for account-linked settings, app configuration, response presets, send records, and related app data.
  • Firebase Remote Config for app configuration and feature behavior.
  • Firebase Cloud Functions for service operations connected to account, data, or purchase workflows.
  • RevenueCat for purchase, subscription, entitlement, and receipt-related services.
  • Google Play Billing or Google Play payment services, and App Store payment providers, for payment processing, subscription management, refunds, chargebacks, and dispute handling.
  • Tally or similar form services for access requests, contact forms, and feedback collection.
  • GitHub Pages for hosting the landing page, policy pages, and related static content.

These third-party services may process data under their own terms and privacy policies.

For Google Analytics for Firebase, Google may process app measurement data as a service provider. More information is available in the Google Privacy Policy and Firebase Privacy and Security documentation.

7. Analytics Collection and Choices

Firebase Analytics is enabled by default when the app is installed and used. PixBridge does not currently provide a separate in-app opt-in step for Analytics.

The Firebase SDK may automatically collect certain events, and PixBridge may collect additional custom events showing whether important feature steps were completed and which allowed category values apply. Analytics collection failure does not block the core Camera-to-API features of the app.

PixBridge does not include direct user-entered information such as email addresses, user names, server URLs, tokens, raw responses, or free-text strings in custom Analytics events. Firebase Analytics may process technical identifiers used to distinguish app installation instances. Analytics information is used for aggregated usage insights and feature improvement.

8. Data Retention

PixBridge may retain account-linked settings, server configurations, response settings, form submissions, and technical records for as long as needed to provide the service, respond to requests, maintain reliability, or meet operational needs.

User-configured external servers may retain images or API data according to their own policies. PixBridge does not control retention practices on servers configured by the user.

When a user deletes their account, PixBridge deletes app usage data stored on PixBridge-controlled systems, such as server addresses, Bearer tokens, response presets, send records, app settings, the Firebase Authentication account, and local cache where supported by the app.

Some limited records may be retained for a period where needed for payment, refund, subscription management, chargeback or dispute handling, security, fraud prevention, or legal obligations. RevenueCat and app store or play store payment providers may also retain purchase and subscription records under their own policies.

Analytics data may be processed according to the configured Google Analytics or Firebase retention settings and Google's applicable retention policies. Account deletion does not necessarily guarantee immediate deletion of all aggregate or non-identifying Analytics data. App instance-based Analytics data may be handled separately under Google Analytics or Firebase settings and policies.

9. Security

PixBridge uses reasonable technical and organizational measures to protect information. No mobile app, network transmission, or external API connection can be guaranteed to be completely secure.

Users should avoid entering long-lived or overly broad Bearer tokens and should rotate or revoke tokens if they suspect exposure or misuse.

10. User Choices and Responsibilities

  • Users choose which server API to connect and what images to send.
  • Users are responsible for having appropriate rights and permissions for images they capture, upload, or process.
  • Users should review token scope, server security, and third-party API policies before connecting PixBridge.
  • Users can request account deletion in the app through Settings > Account > Delete Account.
  • Users may also send a data deletion request to personalplanet.apps@gmail.com.
  • More details are available on the PixBridge Data Deletion page.
  • Users may request assistance or ask about data handling by contacting PixBridge.

Account deletion does not necessarily cancel an active subscription. Users may need to cancel subscriptions through the app store subscription settings or the payment management menu in the app.

Users can stop future app usage data collection by deleting the app. Advertising and privacy settings provided by the operating system or Google account may be managed separately under the relevant platform policies.

11. Children's Privacy

PixBridge is not intended for use by children. Users should not submit information about children or images of children unless they have a lawful basis and appropriate authority to do so.

12. International Availability

PixBridge may be accessed from different regions. Data may be processed through third-party services or infrastructure located outside the user's country or region.

13. Changes to This Policy

PixBridge may update this Privacy Policy as the app, third-party services, business needs, or legal requirements change. The effective date will be updated when material changes are made.

14. Contact

For questions about this Privacy Policy or PixBridge data handling, contact: personalplanet.apps@gmail.com
